Introduction

HTML, or Hypertext Markup Language, is the cornerstone of the whole world wide web. This markup language has been the backbone (ribs, arms, and legs) of webpages for decades. From your personal GeoCities page back in high school to your Myspace account in high school, HTML has been serving the internet to you in an easily read format for a while. You may still remember a lot of the language from personalizing your Myspace with glittery GIFs and tight tiled backgrounds.

A webpage is more than just HTML, of course. Modern day websites employ CSS and Javascript often, but, still, it all starts with the HTML. You make the general cite with HTML and then you build out its style with CSS, just like a skin, and its capabilities with Javascript - kind of like its muscles.

All of this code is stored on a server, and, as you know, you must use a web browser to connect to the website stored on the server. Your browser parses the code, then it constructs the site.

Websites have a lot of moving parts, and rely on many types of code. This particular site - though using HTML and CSS - will focus primarily on HTML.

Basics

There are several elements a website's code should always have. Think of it like the formatting for a research paper or an essay. Each paper, or website, should have headings formatted with specific standards.

In a high school or college class your papers will always have a heading in the upper, left-hand corner of your first page just like this:

Jack Doe
Prof. Jane Doe
Intro to HTML
1 January, 2001

Your HTML code will have a heading as well. It does not contain detailed documentation as this part is not for people to read, but is, instead, intended for the browser. This heading will look like this:

<!DOCTYPE html>

This is not all your code will need, though. You also have to nest all the code into the <html> element. HTML elements will nearly always have a corresponding element closing it off, just like this:

<html>
YOUR CODE HERE
</html>
Elements

Now there has been many references to elements, but what is an element?

The HTML Element page on Wikipedia defines it like this:

"An HTML Element is an individual component of and HTML document or webpage."

That sounds a little vague, so here are a few examples:

  • <body> = The body element defines the body of the webpage. Everything you have been reading is in the body
  • <h1> = First heading
  • <p> = Paragraph

These are the most common elements used on webpages, especially by beginners. This is not a comprehensive list. Many more elements exist, and many have varied forms. The h1 tag, for instance, has slight variations like:

  • <h2>
  • <h3>
  • <h4>
Notice the declination in numbers by the heading tag. The heading tags transform the text nested inside. Text is made bold and the font-size is made large. Now as you decrease the heading number, so will the font-size decrease. Treat it like a sub-heading in an essay.

Remember, many elements have items nested in them. That means an end tag should follow the contents of the element. The <h1> element should be followed up by the </h1> , and the <p> element should be followed by </p> .

Attributes

As mentioned before, when using elements such as <p> and </p> you nest your paragraph inside. That is simple, but more can be done with the element by using attributes.

An attribute is directly associated with the element. Instead of inserting the attribute between the two tags of the element like this:

<p> attribute </p>

you should, instead, insert the attribute inside the first tag itself. Like this:

<p attribute> content </p>

The end tag will not contain anything pertaining to the attribute.

This far part of the syntaxic relationship between attributes and elements has been explained, but what is an attribute? W3schools.com says, "Attributes provide additional information about HTML elements." Simply that means it is a tool that allows elements to have extra skills. Notice the citation for w3schools in this paragraph. It is shown as a link, and that link was made with an attribute. The attribute href="" can be used with the element <a>, also referred to as an anchor, to create a link. A website's URL is added between the quotation marks of the href="" attribute to point to the location the link will take the end-user. It will look like this:

<a href="https://www.w3schools.com/html/html_attributes.asp">

Now if the end-user simply hovers their cursor over the link without clicking on it, the name w3schools.com appears in a box alongside the cursor. This is created by using the attribute title="" where text is written inside the quotation marks of the attribute. The entire element of the link looks like this:

<a href="https://www.w3schools.com/html/html_attributes.asp" title="w3schools.com">W3schools.com</a>

The actual text of the link being clicked on is nested between the anchor elements.

Attributes always effect the contents of the element it is attributed. Here is an attribute that will apply stylistic changes to the element. This attribute is aptly named style="" . Think of this as a text-based version of the font bar on a wordprocessing program. Text-based may sound intimidating at first, but it is really rather simple. No, there is no visual representation to the functions available, and, yes, that means all the functions are memorized. It is actually more simple than one may think. There is more freedom here than scouring through menu after menu to find the tool needed. It has been said, in reference to commandlines and text-based programming, that it is the fastest way to preform a task than any other way.

The style="" attribute will let you change font color, font size, and background color, just to name a few. Take a look at a few examples.

<p style="font-size:36px">The quick brown fox</p>

The quick brown fox

Building off this:

<p style="color: red; font-size:36px;">The quick brown fox</p>

The quick brown fox

Once again, these are just a few examples of different type of attributes. There are too many examples to show here. The w3schools website is an excellent resources for all kinds of attributes available in HTML.

Formatting

Building off of the style attribute from the last section, there are other ways to add stylistic elements to HTML text on a webpage. It is an element, truly. Formatting is a small way to give text unique qualities, compared to the other text surrounding it, by using in-text elements. Notice the word "formatting" in the preceding sentence. The word has been made bold. Returning to the previous example of the word processing program, a button can be selected to transform text into and emboldened state. This same function can be done in HTML with the <b> element. The same was done with the word "is" in the sentence preceding that by using <em> .

Why is this called formatting? It is not like the formatting of a college essay, so, again, why is this called formatting? W3schools says, "HTML uses elements like <b> and <i> for formatting output, bold or italic text." A lot of the elements covered here are on the "micro" side of things, style and formatting primarily format text, not the large scale, or "macro", layout of a page.

Classes

To make changes on the large scale side of the HTML, the <class=""> attribute can be used to edit or transform the multiple elements it is associated. Notice how the heading and paragraph of each section on this webpage are uniform in style. All the headings and paragraphs have the same font-family, font-size, and indentation. The header and paragraph are nested inside a <section> element with the attribute <class=""> , which in practice looks like this:

<section class="main-section">
<p> TEXT </p>
</section>

There is another language commonly used on websites called CSS. The style attribute employees some CSS to change the appearance of HTML. Class attributes are typically used in conjunction with CSS to modify any elements associated with a specific class. Using the CSS style sheet alongside the class attribute multiple attributes can be added to an element with much less clutter. The CSS on this page looks similar to this:

<style>
.main-section{
font-family: courier;
font-size: 16px;
}
</style>

The CSS command that references the associated class it is modifying is a period with no space preceding the name of the class. When <section class="trombone"> is modified using CSS, it is referenced with .trombone{}

IDs

A class attribute is used with CSS to stylize multiple elements, but the ID attribute is used to make large scale changes the contents of a single element. ID attributes are added to elements the same way as class attributes.

<section id="example">

Within the style sheet, though, it is referenced with pound sign, or hashtag, like this:

<style>
#example {
}
</style>

ID attributes can also be used with the class attribute.

<section class="main-section" id="example">

This page uses the ID attribute alongside the class attribute, but not for further stylistic changes. The menu, with links to each section of this document, is linked to each sections ID attribute. This is what allows the link to jump to the section selected.

Each item in the list of options has an href="" attribute associated with it that references its contents, being the ID of each section, then scrolls the page location down to the section selected.
